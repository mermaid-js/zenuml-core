Why ZenUML

Diagrams that live in your codebase

Because the diagram is text, it belongs in the same place as everything else you version, review and ship. Here is what that buys you.

Diagrams as code

Describe a flow in plain text and ZenUML draws it. Edit a line, the diagram updates — no dragging boxes into place.

Version-controllable

Diagrams diff cleanly in Git. Review them in a pull request, revert a bad change, and keep them in sync with the code they document.

No server needed

Rendering happens client-side with a React 19 engine. Your diagram source and data never leave the browser.

Fragments built in

Wrap logic in if/else, opt, while loops and par blocks. Alt, option, loop and parallel frames draw automatically.

Themeable output

The renderer produces clean, styleable SVG, so you can match your brand, your docs, or a dark theme without post-processing.

Embeddable widget

Install @zenuml/core from npm and mount the renderer anywhere — docs sites, dashboards, internal tools, or a Confluence page.

How it works

Three steps, one arrow each

The pipeline is short on purpose: text in, diagram out. Here is the exact path a line of DSL takes.

▸ 01

Write the DSL

Type a message like A->B: request or a call like A.method(). Add fragments, comments and participants as you go.

▸ 02

Parse the text

ZenUML parses your text into an ordered model of participants and messages, resolving who talks to whom and in what sequence.

03

Render the SVG

The React renderer lays out boxes, lifelines and arrows and paints a crisp SVG sequence diagram — instantly, in the browser.

Syntax

The whole language on one card

You already know most of it. The rest fits in a few rows.

A.method()A calls a method on itself (a self-message).
A->B: textA sends a message labelled “text” to B.
A-->B: textA returns a value to B with a dashed arrow.
if(cond){ }Conditional branch — draws an alt fragment.
while(cond){ }Repeat a block — draws a loop fragment.
par { }Run messages in parallel inside one frame.
@Actor UserDeclare a participant and give it an actor icon.
// commentAdd an inline note; == text == draws a divider.

FAQ

Questions, answered

What is ZenUML?

ZenUML is an open-source JavaScript library, published as @zenuml/core, that turns Markdown-style text into sequence diagrams. It renders entirely in the browser, so you never need a diagram server.

How do I create sequence diagrams from text?

Write a few lines of DSL, such as A.method or A->B: message, wrap conditionals in if(cond){…} and loops in while(cond){…}. ZenUML parses the text and renders an SVG sequence diagram instantly.

Do I need a server or an account to render diagrams?

No. ZenUML runs client-side using a React 19 renderer. You install @zenuml/core from npm and render diagrams locally, or use the hosted Live Editor at app.zenuml.com.

Where can I use ZenUML?

It works in Confluence through the Atlassian Marketplace app, in the ZenUML Live Editor, in JetBrains IDEs, in VS Code, in a Chrome extension, and inside Mermaid, which supports zenuml as a diagram type.

Which diagram features does the DSL support?

Messages, self-messages, activations, participant declarations with @Actor, comments, dividers, and combined fragments including alt via if/else, opt, loop via while, and par for parallel work.

Is ZenUML free and open source?

Yes. ZenUML is open source under an MIT-style license and developed in the open at github.com/mermaid-js/zenuml-core. Read the code, file issues, and contribute.
